# Clavus TUI + Push/Pull Sync — Implementation Plan

> **For Hermes:** Use subagent-driven-development to implement this plan task-by-task.

**Goal:** Build a cross-platform Textual TUI that gives Chris and Steven a shared terminal dashboard for cue management + sync, backed by push/pull over Tailscale/SSH.

**Architecture:** Python Textual app (CRUX dark theme) replaces the web companion as the daily driver. Push/pull sync uses the existing `clavus serve` HTTP server with new REST endpoints (`/api/sync/pull`, `/api/sync/push`). The TUI connects to any running Clavus server (Steven connects to Chris's Tailscale IP, Chris connects to localhost).

**Tech Stack:** Python 3.10+, Textual 8.x, httpx (async HTTP), rich (for terminal markup), uvicorn (server), FastAPI (server)

**Key design decisions:**
- TUI and Web companion coexist — both consume the same REST API
- Sync is *project-scoped* — push/pull happens per project, not for the whole store
- Conflict resolution for cues: last-write-wins by timestamp (simple, works for two-person)
- No blob syncing — snapshots stay local to each machine. Only cues and metadata sync
- Platform agnostic: Python stdlib + pip packages, no system deps

**Step 1: Write the pull endpoint**

`GET /api/sync/pull?name={project_name}`

Returns a JSON bundle containing:
```json
{
  "project": { "name": "...", "head": "...", "bpm": 120, "track_count": 14 },
  "cues": [
    { "id": "...", "position": "2:30", "text": "...", "author": "Chris",
      "status": "pending", "timestamp": 1234567890, "replies": [] }
  ],
  "snapshots": [
    { "hash": "61118be0", "message": "Initial import", "timestamp": 1234567890,
      "track_count": 14, "bpm": 120.0 }
  ]
}
```

```python
@app.get("/api/sync/pull")
async def sync_pull(name: str = Query(..., description="Project name to pull")):
    """Pull full project state: cues + snapshot history."""
    try:
        store, proj = _get_project(name)
    except HTTPException:
        return JSONResponse({"error": "Project not found"}, status_code=404)

    # Cues
    cues_store = CueStore(proj.name, store=store)
    all_cues = cues_store.list_cues(CueFilter())
    cues_data = [
        {
            "id": c.id, "position": c.position, "text": c.text,
            "author": c.author, "status": c.status, "timestamp": c.timestamp,
            "track_name": c.track_name,
            "replies": [{"author": r.author, "text": r.text, "timestamp": r.timestamp}
                       for r in (c.replies or [])],
        }
        for c in all_cues
    ]

    # Snapshot history
    history = []
    current = proj.head
    while current:
        snap = store.load_snapshot(current)
        if not snap:
            break
        history.append({
            "hash": snap.hash[:8], "full_hash": snap.hash,
            "timestamp": snap.timestamp, "message": snap.message,
            "track_count": snap.track_count, "bpm": snap.bpm,
            "is_head": current == store.read_ref("HEAD"),
        })
        current = snap.parent

    return {
        "project": {"name": proj.name, "head": proj.head[:8] if proj.head else None, "branch": proj.branch},
        "cues": cues_data,
        "snapshots": history,
    }
```

**Step 2: Write the push endpoint**

`POST /api/sync/push?name={project_name}`

Body: same `{ "cues": [...] }` format from the TUI. The server merges incoming cues into its local store using last-write-wins by timestamp.

```python
@dataclass
class SyncCue:
    id: str; position: str; text: str; author: str
    status: str = "pending"; timestamp: float = 0.0
    track_name: str = ""; snapshot_hash: str = ""
    replies: list = field(default_factory=list)

class SyncPushModel(BaseModel):
    cues: list[SyncCue]
```

```python
@app.post("/api/sync/push")
async def sync_push(data: SyncPushModel, name: str = Query(..., description="Project name")):
    """Push cues from a remote client. Merges using last-write-wins by timestamp."""
    try:
        store, proj = _get_project(name)
    except HTTPException:
        return JSONResponse({"error": "Project not found"}, status_code=404)

    cues_store = CueStore(proj.name, store=store)
    existing = {c.id: c for c in cues_store.list_cues(CueFilter())}

    merged = 0
    for incoming in data.cues:
        existing_cue = existing.get(incoming.id)
        if existing_cue and existing_cue.timestamp >= incoming.timestamp:
            continue  # existing is newer, skip
        # Add or update
        cues_store._write_cue(incoming)  # Need to expose write method
        merged += 1

    return {"status": "ok", "merged": merged}
```

**Step 3: Add `_write_cue` method to CueStore**

The CueStore needs a direct write method for sync (currently cues are added via `add_cue` which generates an ID):

```python
# In cues.py, add to CueStore:
def import_cue(self, cue: Cue) -> None:
    """Import a cue from sync (assumes ID is already set)."""
    cues = self._load_cues()
    cues[cue.id] = asdict(cue)
    self._save_cues(cues)

def import_reply(self, cue_id: str, reply: CueReply) -> bool:
    """Import a reply from sync."""
    cues = self._load_cues()
    if cue_id not in cues:
        return False
    existing = cues[cue_id]
    existing_replies = [CueReply(**r) for r in existing.get("replies", [])]
    # Only add if not duplicate (match by timestamp + author)
    dup = any(r.timestamp == reply.timestamp and r.author == reply.author for r in existing_replies)
    if not dup:
        existing_replies.append(reply)
        existing["replies"] = [asdict(r) for r in existing_replies]
        cues[cue_id] = existing
        self._save_cues(cues)
    return True
```

    """A single cue card showing position, text, replies, and status."""

## Phase 4: Client-side sync (two-way)

### Task 7: Local cue tracking for push

**Objective:** The TUI needs to track which of its local cues have been pushed vs pulled, so the push doesn't send back cues it just received.

**Approach:** Keep a set of "pushed cue IDs" in memory. When pulling from server, add all received IDs to the pushed set. When pushing, only send cues whose IDs aren't in the pushed set (i.e., locally created since last sync).

```python
class ClavusTUI(App):
    def __init__(self, ...):
        ...
        self._pushed_ids: set[str] = set()
    
    async def do_pull(self):
        bundle = await self.client.pull(self.project_name)
        self._pushed_ids.update(c.id for c in bundle.cues)
        ...
    
    async def do_sync(self):
        new_cues = [c for c in self._local_cues if c.id not in self._pushed_ids]
        if new_cues:
            result = await self.client.push(self.project_name, new_cues)
            self._pushed_ids.update(c.id for c in new_cues)
        await self.do_pull()  # Get any remote changes
```

## Risks & Tradeoffs

1. **Textual on Windows:** Textual works on Windows via Windows Terminal. Steven needs Windows Terminal installed (not CMD). Fallback: the web companion still works for him.
2. **Textual on older Python:** Requires Python 3.10+. Both macOS and Windows 11 ships newer.
3. **Sync conflicts:** Last-write-wins by timestamp is naive but fine for two-person cue editing. If conflicts become an issue, switch to CRDT.
4. **No blob sync:** Snapshots stay local. This means Steven can see "Snapshot 61118be0" in the history but can't diff against it locally. That's fine — snapshot sync is Phase 2.
5. **Server must be running:** The TUI is a *client* — it needs a `clavus serve` running somewhere. Chris keeps his running (it's already his dev machine). Steven connects to Chris's Tailscale IP.
